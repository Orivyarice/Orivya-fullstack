2. Payment Gateway (GPay / PhonePe / UPI)

All in one via Razorpay
💡 Razorpay supports: Google Pay, PhonePe, Paytm, BHIM UPI, Credit/Debit Cards, Net Banking — all in ONE integration.
Step 1 — Create Razorpay Account
1. Go to https://razorpay.com → Sign Up
2. Complete KYC with your business documents (FSSAI + bank details)
3. Go to Dashboard → Settings → API Keys → Generate Key
4. You get: Key ID (rzp_live_xxx) and Key Secret
Step 2 — Add to application.properties
razorpay.key.id=rzp_live_YourKeyIDHere razorpay.key.secret=YourKeySecretHere
Step 3 — Add Razorpay SDK to pom.xml
<dependency> <groupId>com.razorpay</groupId> <artifactId>razorpay-java</artifactId> <version>1.4.5</version> </dependency>
Step 4 — Frontend: Open Razorpay Checkout
// In checkout page, after selecting Razorpay: async function openRazorpay(orderData) { const options = { key: orderData.razorpayKeyId, amount: orderData.amount, // in paise currency: 'INR', name: 'Orivya Rice', description: 'Premium Rice Order', image: 'images/logo.png', handler: async function(response) { // Payment success — verify on backend const result = await fetch('/api/payments/verify', { method: 'POST', headers: getHeaders(), body: JSON.stringify({ razorpay_payment_id: response.razorpay_payment_id, razorpay_order_id: response.razorpay_order_id, razorpay_signature: response.razorpay_signature, deliveryAddress: document.getElementById('checkAddress').value }) }); // Show success }, prefill: { name: orderData.customerName, email: orderData.customerEmail, contact: orderData.customerPhone }, theme: { color: '#1a5c2a' } }; const rzp = new Razorpay(options); rzp.open(); }
✅ After this, customers can pay via Google Pay, PhonePe, Paytm, BHIM, Cards, Net Banking — all from one popup.

4. Docker — Run on Any Server

Containerize the entire application
Docker Setup Structure:

orivya-mysql container  ↔  orivya-backend container  ↔  orivya-frontend (nginx) container
All 3 connected via orivya-network bridge
Files Created (in your project):
orivya-fullstack/ ├── docker-compose.yml ← Run all 3 containers together ├── backend/ │ └── Dockerfile ← Build Spring Boot JAR └── frontend/ ├── Dockerfile ← Nginx serving HTML/CSS/JS └── nginx.conf ← Proxy /api calls to backend
Local Docker Commands
# Start everything (MySQL + Backend + Frontend) docker-compose up -d # See running containers docker-compose ps # See logs docker-compose logs -f orivya-backend # Stop everything docker-compose down # Rebuild after code changes docker-compose up -d --build After running: open http://localhost — your full app is live!

6. Terraform — Infrastructure as Code

Create AWS resources automatically
File created: devops/terraform/main.tf
Creates: VPC, EC2, RDS MySQL, S3, ECR automatically
Commands to Run
# Install Terraform first: https://terraform.io/downloads cd devops/terraform # Initialize (downloads AWS provider) terraform init # Preview what will be created terraform plan -var="db_password=OrivyaDB@2024" # Create all AWS resources (takes ~5 minutes) terraform apply -var="db_password=OrivyaDB@2024" # At the end, you'll see: # app_server_ip = "13.xxx.xxx.xxx" # rds_endpoint = "orivya-mysql.xxx.rds.amazonaws.com" # Destroy everything when done (saves cost) terraform destroy -var="db_password=OrivyaDB@2024"

7. Kubernetes on AWS EKS

Auto-scaling, self-healing, zero-downtime
Create EKS Cluster (AWS Managed K8s)
# Install eksctl curl --silent --location "https://github.com/weaveworks/eksctl/releases/latest/download/eksctl_Linux_amd64.tar.gz" | tar xz sudo mv eksctl /usr/local/bin # Create EKS cluster (takes ~15 minutes) eksctl create cluster \ --name orivya-cluster \ --region ap-south-1 \ --nodegroup-name orivya-nodes \ --node-type t3.medium \ --nodes 2 \ --nodes-min 1 \ --nodes-max 4 # Deploy the application kubectl apply -f devops/kubernetes/k8s-deployment.yml # Watch pods starting kubectl get pods -n orivya -w # Get the public URL kubectl get service orivya-frontend-service -n orivya

9. Prometheus + Grafana Monitoring

Monitor your app in real time
Step 1 — Add Spring Boot Actuator to pom.xml
<dependency> <groupId>org.springframework.boot</groupId> <artifactId>spring-boot-starter-actuator</artifactId> </dependency> <dependency> <groupId>io.micrometer</groupId> <artifactId>micrometer-registry-prometheus</artifactId> </dependency>
Step 2 — Add to application.properties
management.endpoints.web.exposure.include=health,info,prometheus,metrics management.endpoint.health.show-details=always management.metrics.export.prometheus.enabled=true Then visit: http://localhost:8080/actuator/prometheus to see metrics.
Step 3 — Start Prometheus + Grafana (already in docker-compose.yml)
docker-compose up -d prometheus grafana Prometheus UI: http://localhost:9090
Grafana UI: http://localhost:3000 (admin / admin123)
Step 4 — Import Grafana Dashboard
In Grafana: Dashboards → Import → Enter ID 12900 (Spring Boot Dashboard)
You'll see: Request rate, Response time, JVM memory, DB connections, Error rate — all in real time.

11. Ansible Server Automation

Setup new servers automatically
Create inventory.ini with your server IP
[orivya_servers] 13.xxx.xxx.xxx ansible_user=ubuntu ansible_ssh_private_key_file=~/.ssh/id_rsa
Run the playbook
# Test connection ansible -i inventory.ini orivya_servers -m ping # Run full setup (installs Docker, clones repo, starts app) ansible-playbook -i inventory.ini devops/ansible/setup-server.yml This automatically installs Docker, sets up firewall, clones your code, and starts the app.

Full picture of your DevOps setup

    GitHub Actions CI/CD

    Monitoring Stack

    └── Alerts → Email/Slack if issues
