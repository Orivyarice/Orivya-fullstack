Install Prerequisites

Software you need before starting
☕
Install Java 17 (JDK)
Download from: https://adoptium.net
Choose: Eclipse Temurin 17 (LTS) → Download installer → Install
Verify installation: java -version You should see: openjdk version "17.x.x"
🏗️
Install Maven
Download from: https://maven.apache.org/download.cgi
Extract the zip → Add bin folder to PATH environment variable
Verify installation: mvn -version
💡 Easier option: Use IntelliJ IDEA or VS Code with Java extensions — Maven is bundled automatically.
🗄️
Install MySQL 8.0
Download from: https://dev.mysql.com/downloads/installer/
Install MySQL Community Server + MySQL Workbench
During setup, set a root password — remember it for Step 3!
Verify by opening MySQL Workbench and connecting with root.
💻
Install VS Code (for frontend)
Download from: https://code.visualstudio.com
Install the Live Server extension (by Ritwick Dey) to run the frontend.
2

Set Up MySQL Database

Create the database before starting the backend
🗄️
Create the database
Open MySQL Workbench → Connect → Open a new query tab → Run: CREATE DATABASE orivya_db; USE orivya_db; SELECT 'Database created!' as message;
✅ You should see "Database created!" in the output.
📊
Tables are auto-created by Spring Boot
You do NOT need to create tables manually.
When you start the Spring Boot app for the first time, JPA (Hibernate) will automatically create all tables:
users, products, orders, order_items, cart_items
This is controlled by spring.jpa.hibernate.ddl-auto=update in application.properties
3

Configure the Backend

Update application.properties with your credentials
⚙️
Edit application.properties
Open: backend/src/main/resources/application.properties
Change only these 2 lines: # Change 'your_mysql_password' to your actual MySQL root password spring.datasource.password=your_mysql_password # Optional: change port if 8080 is busy server.port=8080
⚠️ The JWT secret and other settings are already configured. Only change the MySQL password.
4

Run the Spring Boot Backend

Start the REST API server
▶️
Option A — Using Terminal (Command Line)
Open terminal/command prompt → navigate to the backend folder: cd orivya-fullstack/backend mvn spring-boot:run Wait for the startup message. You'll see: 🌾 Orivya Rice Backend is running at http://localhost:8080 Started OrivyaApplication in 4.532 seconds
🧠
Option B — Using IntelliJ IDEA (Recommended for beginners)
1. Open IntelliJ IDEA → File → Open → select the backend folder
2. Wait for Maven to download dependencies (first time takes 2-5 minutes)
3. Find OrivyaApplication.java → right-click → Run 'OrivyaApplication'
4. See the green "Started" message in the console.
🔍
Verify the backend is working
Open your browser and go to: http://localhost:8080/api/products You should see: {"success":true,"message":"Products fetched successfully","data":{"content":[]...}}
✅ Backend is running! (Empty products array is correct — we'll add data next)

Common Errors & Fixes

Solutions to problems you might face
🔌
Cannot connect to MySQL
Error: Access denied for user 'root'@'localhost'
Fix: Open application.properties → update spring.datasource.password to your MySQL root password.
🌐
CORS Error in browser console
Error: Access to fetch at 'http://localhost:8080' from origin 'http://127.0.0.1:5500' has been blocked
Fix: In SecurityConfig.java, the allowedOriginPatterns includes * which should allow all. If still failing, restart the backend.
🔒
401 Unauthorized on cart/order APIs
Cause: JWT token is missing or expired.
Fix: Make sure you are logged in. Open browser DevTools → Application → Local Storage → check orivya_token exists. If not, login again.
☕
Port 8080 already in use
Error: Web server failed to start. Port 8080 was already in use.
Fix: Change server.port=8081 in application.properties, then update API_BASE in frontend/js/api.js to http://localhost:8081/api.
📦
Maven dependencies not downloading
Fix: Make sure you have internet connection, then run: mvn dependency:resolve Or in IntelliJ: right-click pom.xml → Maven → Reload Project.
